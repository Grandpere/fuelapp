# Backlog structure

## Why this structure
- Sprint files stay easy to scan.
- Ticket files keep implementation details and acceptance criteria clear.
- IDs are stable and usable in PR titles, branches and commits.

## Conventions
- Sprint file: `sprint-01.md`, `sprint-02.md`, etc.
- Ticket file: `SP1-001-short-name.md`.
- Status values: `todo`, `in_progress`, `blocked`, `done`.
- Priority values: `P0`, `P1`, `P2`.

## Ticket template
Use this template for new tickets:

```md
# SPX-YYY - Title
```
