# SP12-004 - Security observability and incident runbook hardening

## Context
Security events are logged, but detection and triage should be formalized.

## Scope
- Define critical security alerts and thresholds (auth failures, suspicious API patterns, admin-sensitive actions).
- Add security-focused dashboards/queries in local observability stack.
- Expand incident response runbook with repeatable diagnostic steps.

## Out of scope
- Managed SOC integration.

## Acceptance criteria
- Security incident checklist is documented and actionable.
- Key alerts can be verified end-to-end locally.

## Dependencies
- Sprint 10 observability baseline.
