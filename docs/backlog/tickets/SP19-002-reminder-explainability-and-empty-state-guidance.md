# SP19-002 - Reminder explainability and empty-state guidance

## Why
Even with reminder cards and rule CRUD, the maintenance dashboard still needs to explain why reminders are missing or not yet due so users can trust the feature.

## Scope
- Improve empty states around reminder rules and triggered reminders.
- Explain the main blocking cases: no rule, rule not due yet, or insufficient mileage/date context.
- Keep the work front-office focused; admin parity can stay deferred.
