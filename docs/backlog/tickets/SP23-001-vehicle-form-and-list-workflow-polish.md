# SP23-001 - Vehicle form and list workflow polish

## Why
The vehicle hub is already strong, but the base list and form flows still lose context too easily and require extra navigation for common next steps.

## Expected outcome
- Vehicle create/edit flows keep a safe return path instead of always dropping back on the generic list.
- `/ui/vehicles` exposes the most useful next actions directly from each row.
- The list and form together feel like a smoother front-office workflow instead of isolated screens.

## Notes
- Front-office only for now.
- No admin parity needed at this stage.
