# SP24-003 - Front dashboard cohesion pass

## Why
The main front-office hubs are now richer, but some empty states still feel like dead ends instead of helping users move to the next useful action.

## Expected outcome
- Vehicle, station, maintenance, and receipt hubs expose obvious next-step actions when they are empty or only partially populated.
- Empty states keep users inside the current front-office workflow instead of forcing them to guess where to go next.
- Cross-screen navigation feels more cohesive even before data starts accumulating.

## Notes
- Front-office only for now.
- No admin parity needed at this stage.
